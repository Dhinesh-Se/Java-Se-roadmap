↓ PROCEED TO DESIGN PATTERNS
Phase 02
GoF Design Patterns (23 Patterns)
⏱ 5–7 weeks
▸ CREATIONAL — Object Creation Mechanisms
Singleton
Creational
One instance, global access point. Thread-safe with double-checked locking or enum.
Factory Method
Creational
Delegate instantiation to subclasses. Decouples creation from usage.
Abstract Factory
Creational
Create families of related objects without specifying concrete classes.
Builder
Creational
Construct complex objects step by step. Fluent API in Java (Lombok @Builder).
Prototype
Creational
Clone objects instead of new instantiation. Deep vs shallow copy.
Object Pool
Creational
Reuse expensive objects (connections, threads). Used in DB connection pools.
▸ STRUCTURAL — Class & Object Composition
Adapter
Structural
Translate one interface to another. Bridging legacy and new code.
Bridge
Structural
Separate abstraction from implementation so both can vary independently.
Composite
Structural
Tree structures where individual objects and groups are treated uniformly.
Decorator
Structural
Add behavior to objects dynamically. Java I/O streams are a prime example.
Facade
Structural
Simple interface to a complex subsystem. Used in service layers.
Flyweight
Structural
Share state among many fine-grained objects. String pool in Java.
Proxy
Structural
Surrogate controls access. Lazy loading, caching, logging, Spring AOP.
▸ BEHAVIORAL — Object Communication & Responsibilities
Observer
Behavioral
Notify dependents of state changes. Events, Kafka consumers, Spring Events.
Strategy
Behavioral
Define a family of algorithms, make them interchangeable at runtime.
Command
Behavioral
Encapsulate requests as objects. Enables undo/redo, queuing, macros.
Template Method
Behavioral
Define skeleton in base class, let subclasses fill in specific steps.
Iterator
Behavioral
Sequential access without exposing underlying structure. Java Iterable.
State
Behavioral
Object behavior changes with internal state. FSM implementation.
Chain of Responsibility
Behavioral
Pass requests along handler chain. Filter chains in Servlet/Spring.
Mediator
Behavioral
Centralize communication between objects. Message brokers, chat rooms.
Memento
Behavioral
Save/restore object state without exposing internals. Version history.
Visitor
Behavioral
Add operations to object structure without modifying classes. AST traversal.
↓ ADVANCE TO ARCHITECTURAL PATTERNS
Phase 03
Architectural & Enterprise Patterns
⏱ 4–5 weeks
🏗️
Layered / N-Tier Architecture
Separate concerns into Presentation, Business, Data Access layers. Foundation of most Java apps.
Controller Service Layer Repository/DAO Spring MVC
🧱
MVC / MVP / MVVM
Separate UI logic from business logic. MVC is dominant in Spring and Java EE.
Model View Controller Presenter ViewModel
🗃️
Repository & DAO Pattern
Abstract data persistence. Use interfaces for testability. Spring Data JPA automates this.
JPA Hibernate Unit of Work Spring Data
🔧
Dependency Injection / IoC
Invert control — wire dependencies externally. Core of Spring Framework.
Spring IoC Container @Autowired @Bean Constructor Injection
📬
Event-Driven Architecture
Components communicate via events, not direct calls. Loose coupling for scale.
ApplicationEvent Kafka RabbitMQ Async Processing
🌀
CQRS & Event Sourcing
Separate read/write models. Store state as a sequence of events, not current state.
Command Query Event Store Axon Framework

Phase 04
System Design Fundamentals
⏱ 5–6 weeks
⚖️
Scalability Concepts
Design systems that grow horizontally and vertically without breaking.
Horizontal Scaling Load Balancing Sharding Replication Partitioning

Database Design & Storage
SQL vs NoSQL trade-offs, indexing strategies, normalization, and query optimization.
RDBMS NoSQL (Mongo, Cassandra) Indexing ACID vs BASE CAP Theorem

Redis Memcached Cache-Aside Write-Through TTL & Eviction

Build APIs that are intuitive, versioned, and performant. REST is dominant in Java microservices.
REST Principles gRPC / Protobuf OpenAPI/Swagger Rate Limiting Pagination
📨

Asynchronous communication between services. The backbone of scalable systems.
Apache Kafka RabbitMQ Topics & Partitions Consumer Groups Backpressure
🔒
Security in Systems
Authentication, authorization, and data protection at scale.
OAuth2 / JWT Spring Security TLS/HTTPS RBAC Secret Management
